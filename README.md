# 운동 칼로리 추정기 (Workout Calorie Estimator)

> 사용자 입력 기반 운동 소모 칼로리 추정 및 일일 칼로리 밸런스 관리 웹 서비스

## 프로젝트 개요

단순한 "운동 종류 선택 → 칼로리 표시" 방식이 아닌, **실제 수행 데이터(속도, 중량, 세트, 반복수)**와 **개인 신체 정보**를 기반으로 유산소 및 웨이트 소모 칼로리를 추정합니다.  
식단 기록과 결합하여 하루 단위 칼로리 밸런스 및 체중 방향을 시각화합니다.

---

## 주요 기능

### 운동 칼로리 계산
- **유산소**: Compendium of Physical Activities MET 기반 계산
  - 속도·시간·경사도 입력, 8km/h 이상 시 달리기 MET 자동 적용
- **웨이트**: Lytle & Lambert (2019) 회귀방정식으로 세션 총 kcal를 추정한 후, 운동별 기여도(볼륨 × 운동군 가중치 × 종목 보정 × 휴식 보정)에 따라 분배하는 방식으로 산출
  - 체지방량 입력 시 더 좁은 오차 범위 적용 (정밀 모드), 미입력 시 더 넓은 오차 범위 적용 (추정 모드) — 오차 범위는 프로젝트 내부 정책값
  - 어시스트 머신(풀업/딥스) 실제 부하 역산 처리 (실제 부하 = 체중 - 보조중량)

### 신체 지표 시각화
- BMI 게이지 바 (저체중 / 정상 / 과체중 / 경도비만 / 고도비만)
- 기초대사량(BMR), 적정 체중, 예상 유지 칼로리 자동 계산

### 칼로리 밸런스
- 식단 섭취 칼로리 기록 (음식명 / 칼로리 / 날짜 / 메모)
- **운동 기준 밸런스**: 섭취 − 운동 소모
- **하루 순 밸런스**: 섭취 − BMR − 운동 소모  
  (운동 소모뿐 아니라 기초대사량까지 반영한 하루 에너지 수지)
- **체중 방향 표시**: 체중 감소 방향 / 유지 방향 / 체중 증가 방향

### 기록 관리
- localStorage 기반 날짜별 운동·식단 기록 저장 (최근 365일)
- 과거/미래 날짜 운동 입력 가능
- 월별 달력 (기록 있는 날 표시)
- 최근 2주 칼로리 추이 차트 (운동 소모 / 섭취 / 순 밸런스)

## 계산 근거

### 유산소
- Ainsworth et al., *Compendium of Physical Activities* (2011)
- 공식: MET × 체중(kg) × 시간(h)

### 웨이트
- Lytle & Lambert (2019), *"Predicting Energy Expenditure of Resistance Exercise"*  
  R²=0.773, SEE=28.5 kcal  
- 연구식으로 세션 총 kcal를 추정한 후, 운동군 가중치·종목 보정·휴식 보정 등 프로젝트 내부 보정계수를 적용하여 운동별로 분배

### 신체 지표
- BMR: Mifflin-St Jeor 공식
- 적정 체중: 브로카 변법
- 체지방량 미입력 시 추정: Deurenberg 공식

## 개발 배경

기존 운동 앱들이 "낮음/보통/높음" 같은 주관적 선택지로 칼로리를 추정하는 방식에 의문을 가져, 실제 수행한 운동 데이터를 직접 입력받아 연구 기반 계산식과 프로젝트 내부 보정 로직을 결합한 서비스를 기획했습니다.  
기획, 설계, 기능 검증, 테스트 전 과정을 직접 주도했으며, AI 도구를 활용하여 구현했습니다.

## 한계 및 주의사항

- 본 서비스의 칼로리 값은 참고용 추정치이며, 의료적 측정치가 아닙니다.
- 개인의 생리적 특성, 운동 수행 방식, 체력 수준에 따라 실제 소모 칼로리와 차이가 있을 수 있습니다.
- 웨이트 운동의 경우 연구식(Lytle et al.)과 프로젝트 내부 보정계수를 함께 사용하며, 운동별 수치는 세션 총 추정치를 상대 기여도로 분배한 값입니다.
- 런닝머신 등 기기의 표시 칼로리와 차이가 발생할 수 있습니다. 기기마다 칼로리 계산 방식이 표준화되어 있지 않기 때문입니다.
